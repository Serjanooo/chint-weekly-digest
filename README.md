# CHINT Russia Weekly Digest

Локальная программа собирает новости за последние семь полных дней, отбирает с помощью Codex десять наиболее релевантных материалов и создает Word-документ в стиле Telegram-дайджестов CHINT Russia.

Программа не использует API сторонних нейросетей. Единственный LLM-вызов выполняется через установленный и авторизованный `codex exec`, поэтому расходуются лимиты вашей подписки Codex. RSS и Google News RSS используются только как источники ссылок и фактов, без API-ключей.

## Что получается

- 10 актуальных новостей со ссылками, зашитыми в смысловые глаголы;
- прямые ссылки на публикации СМИ или первоисточник — адреса Google News в результат не допускаются;
- новость о CHINT в России, если такая публикация была на выбранной неделе;
- собственный инфоповод CHINT Russia из публичного Telegram-канала, если внешних CHINT-новостей нет;
- готовый текст поста с маркером `🔵` и хештегом `#CHINT_Новости`;
- `.docx` в папке `outputs/`;
- JSON с теми же данными для повторной верстки без нового вызова Codex.

## Установка

Нужны Python 3.10+, Microsoft Word или LibreOffice для просмотра результата и установленный Codex CLI с выполненным входом. Программа работает на macOS и Windows.

```bash
cd /path/to/Digest
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"
codex --version
```

## Запуск

### Двойным кликом на macOS

Откройте папку проекта в Finder и дважды нажмите `Запустить дайджест.command`. Скрипт сам подготовит окружение при первом запуске, сформирует выпуск и откроет готовый Word-файл. Окно Terminal можно закрыть после сообщения об успешном завершении.

### Двойным кликом на Windows

Откройте папку проекта в Проводнике и дважды нажмите `Запустить дайджест Windows.bat`. Скрипт запустит PowerShell, сам создаст `.venv`, установит зависимости, сформирует выпуск и откроет готовый Word-файл.

Перед первым запуском на Windows проверьте:

- установлен Python 3.10+ с включённым пунктом `Add python.exe to PATH`;
- установлен и авторизован Codex CLI;
- команда `codex --version` работает в PowerShell.

Если Codex CLI установлен, но команда `codex` не видна в PATH, укажите путь вручную:

```powershell
$env:CODEX_BIN = "C:\Path\To\codex.cmd"
```

### Из терминала

Обычный запуск берет семь полных календарных дней до сегодняшнего дня:

```bash
chint-digest weekly
```

Точный период можно задать явно (обе границы включены):

```bash
chint-digest weekly --from 2026-06-15 --to 2026-06-21
```

Полезные варианты:

```bash
# Только собрать и оценить кандидатов, без расхода лимита Codex
chint-digest collect --from 2026-06-15 --to 2026-06-21

# Сверстать DOCX повторно из уже созданного JSON, без расхода лимита
chint-digest render outputs/digest_2026-06-15_2026-06-21.json

# Обновить профиль при замене обучающего документа
chint-digest init-profile --force
```

В Windows те же команды можно запускать из PowerShell после активации окружения:

```powershell
.\.venv\Scripts\Activate.ps1
python -m digest.cli weekly
```

`weekly` показывает этапы работы и в конце печатает пути к `.docx` и `.json`. Если новостей мало, программа завершится с понятной ошибкой и сохранит собранные материалы для диагностики.

## Как устроено одноразовое обучение

Готовый профиль находится в `profile/style_profile.json`. В нем записан SHA-256 исходного документа. Пока файл `CHINT_Russia_новостные_дайджесты_2025-2026.docx` не меняется, повторный анализ не запускается. Команда `init-profile` нужна только после обновления корпуса; она один раз вызывает Codex и перезаписывает профиль.

Профиль хранит тематические приоритеты, композицию выпуска, тон, нежелательные приемы и примеры прежнего стиля. Сам исходный Word-файл не отправляется новостным сайтам.

## Отбор и достоверность

1. Сборщик читает прямые RSS крупных изданий и поисковые RSS-запросы за выбранные даты.
2. Поисковые ссылки Google News раскрываются до URL оригинального СМИ. Карточки без доступной прямой ссылки или с доменом вне проверенного списка отбрасываются.
3. Материалы вне периода отбрасываются, похожие заголовки объединяются.
4. Редакционный фильтр до Codex снимает новости о Meta/Facebook/Instagram, политические сюжеты, аварийную и криминальную хронику, товарные страницы и другие материалы из стоп-списков.
5. Локальный тематический скоринг поднимает энергетику, ЦОД, ИИ, технологии, связи России с Китаем, внешние новости CHINT и собственные инфоповоды CHINT Russia.
6. Codex выбирает ровно десять разных сюжетов и пишет только по фактам из карточек кандидатов.
7. Программа отдельно ищет любое упоминание CHINT в новостях Elec.ru, RusCable.Ru и «РБК Компании». Если публикация найдена, она обязательно попадает в десятку; если нет, программа смотрит публичный канал CHINT Russia за расширенное окно и может включить собственный пост.
8. Если внешней CHINT-новости нет, Word-документ получает редакционную пометку в конце даже при включённом собственном CHINT-посте.
9. Хотя бы одна из десяти новостей — лёгкий технологический сюжет в духе прежних выпусков.
10. Ссылка на источник в Word зашивается в смысловой глагол внутри новости, как в опубликованных примерах.
11. После генерации программа дополнительно проверяет, что в тексте нет названий сторонних компаний из стоп-списка, политических маркеров и запрещённых организаций.

Перед публикацией редактору все равно следует открыть восемь ссылок: новостные данные могут исправляться источником после выхода.

## Настройка

- `config/sources.json` — RSS, поисковые запросы, веса источников, собственный Telegram-канал, стоп-слова и редакционная политика.
- `profile/style_profile.json` — выученный стиль.
- `prompts/generate_digest.md` — редакционное задание Codex.
- `schemas/digest.schema.json` — строгий формат результата.
- `Запустить дайджест.command` — запуск двойным кликом на macOS.
- `Запустить дайджест Windows.bat` и `scripts/run_digest_windows.ps1` — запуск двойным кликом на Windows.

Чтобы добавить источник, внесите его RSS в `direct_feeds`. Чтобы расширить запреты, правьте блок `editorial_policy`: `blocked_organization_terms`, `political_terms`, `incident_terms` и `other_company_terms`. Символ `*` в конце слова означает поиск по началу слова, например `санкц*`.

Собственные посты CHINT берутся из `own_channel`. Параметр `lookback_days` расширяет окно назад от начала недели дайджеста, чтобы редактор мог выбрать свежий, но не строго недельный инфоповод.

```bash
pytest

```

## Ограничения

- Некоторые сайты публикуют в RSS сокращенный текст или блокируют загрузку статьи. В таком случае Codex получает заголовок и RSS-аннотацию и обязан писать только подтвержденные ими факты.
- Google News RSS может вернуть ссылку-посредник; такие карточки раскрываются до прямого URL СМИ, а если раскрыть их нельзя, они отбрасываются.
- Списки запрещённых организаций и нежелательных компаний в `config/sources.json` — редакционный фильтр, а не юридическая база данных. При необходимости дополняйте их вручную.
